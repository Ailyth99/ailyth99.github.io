関数

sceDevctl
デバイスへの特殊操作
形式

#include <sifdev.h>
int sceDevctl(
	const char *name,
	int cmd,
	void *arg,
	unsigned int arglen,
	void *bufp,
	unsigned int buflen
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 name   デバイス名(dev9x:)。 
 cmd   操作コマンド。次のいずれかの定数で指定します。 
 DDIOC_MODEL 
 DDIOC_OFF 
 arg   コマンドに与える引数。cmd に依存します。 
 arglen   arg のサイズ 
 bufp   コマンドから受け取る引数。cmd に依存します。 
 buflen   bufp のサイズ 

返り値

成功の場合、cmd に依存した値を返します。
エラーの場合、各コマンドは以下の共通エラーを返します。

 値   マクロ   結果 
 -19   -ENODEV   指定されたデバイスが存在しない 
 -24   -EMFILE   オープンできるディスクリプタの最大数に達している 


解説

デバイスに対して特殊操作を行います。各cmd については、「devctlコマンド一覧」を参照してください。デバイス名は、dev9 ではなく、dev9x です。間違わないようにしてください。
例)
sceDevctl ("dev9x:",DDIOC_OFF, NULL, 0, NULL, 0);


devctlコマンド

DDIOC_MODEL
デバイスのタイプを判断
引数

 arg   予約。NULL を指定してください。 
 arglen   arg のサイズ。 
 bufp   予約。NULL を指定してください。 
 buflen   bufp のサイズ。 

返り値

デバイスが、PC Card タイプの場合、0 を、ハードディスクドライブ(EXPANTION BAYタイプ)の場合、1 を返します。

解説

デバイスが、PC Card タイプかEXPANTION BAYタイプかを判断します。通常、アプリケーションがこの操作を行う必要はありません。

DDIOC_OFF
デバイスの電源 OFF
引数

 arg   予約。NULL を指定してください。 
 arglen   arg のサイズ。 
 bufp   予約。NULL を指定してください。 
 buflen   bufp のサイズ。 

返り値

成功の場合、0 を返します。

解説

dev9 デバイス全体の電源を切ります。
Release2.5.2までは、ハードディスクドライブとHDDイーサネットのどちらが接続されているかで電源OFF処理のコマンドが異なりましたが、Release2.5.3のdev9で機能を統一したため、どちらの場合でもこのコマンドを使用してください。
なお、HDIOC_DEV9OFFコマンドも互換性のために残されています。

詳細は、ネットワークライブラリ概要(inet)およびPlayStation File System概要(pfs)の「電源OFF処理」を参照して下さい。
