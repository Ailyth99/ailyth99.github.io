構造

typedef struct sceNetcnfifArg {
	int data;
	int f_no_decode;
	int type;
	int addr;
	char fname[256];
	char usr_name[256];
	char new_usr_name[256];
} sceNetcnfifArg_t;
メンバ

 data   その他の引数及びその他の返り値 
 f_no_decode   no_decodeフラグ 
 type   あなたのネットワーク設定ファイルの種類 
 addr   EE側の受信領域のアドレス及びIOP側のアドレスの返り値

 usr_name   設定名 
 new_usr_name   新しい設定名 

解説

共通ネットワーク設定インタフェースライブラリの引数及びその返り値を定義します。いくつかのメンバについてはその内容が定義されています。

f_no_decodeメンバには以下の値のいずれかが設定されます。
 値   マクロ   意味 
 0   sceNetcnfifArg_no_decode_off   あなたのネットワーク設定ファイルの 
 符号化・復号化を行う 
 1   sceNetcnfifArg_no_decode_on   あなたのネットワーク設定ファイルの 
 符号化・復号化を行わない 


typeメンバには以下の値のいずれかが設定されます。
 値   マクロ   意味 
 0   sceNetcnfifArg_type_net   組み合わせ 
 1   sceNetcnfifArg_type_ifc   接続プロバイダ設定 
 2   sceNetcnfifArg_type_dev   接続機器設定 


fnameメンバにはメモリーカード(PS2)及びハードディスクドライブの設定管理ファイルパスを設定する場合、以下のいずれかの文字列が設定されます。(?はデバイスの接続位置によって変わります)
 デバイス   文字列 
 メモリーカード(PS2)   mc?:/BWNETCNF/BWNETCNF 
 ハードディスクドライブ   pfs?:/etc/network/net.db

解説

あなたのネットワーク設定ファイルに設定する内容を定義します。いくつかのメンバについてはその内容が定義されています。文字列を指定するメンバについては設定を無効とする場合は全ての領域を’\0’にすることにより可能となります。
文字列メンバの設定内容や以下の表の意味の詳細については、共通ネットワーク設定ライブラリ(netcnf)のoverviewを参照してください。
追加ATコマンドには共通ネットワーク設定ライブラリで提供されるsceNetCnfConvA2S()関数が変換可能なATコマンド、および共通ネットワーク設定ライブラリ概要（overview）で定義されるチャットスクリプトを入力可能とします。

解説

SIF RPC処理をノンブロックで行います。処理の終了はsceNetcnfifCheck()で検出する必要があります。この関数を実行するためには、以下の手順で前処理を行う必要があります。

  1. IOPワークエリアの確保。(sceNetcnfifAllocWorkarea())
  2. IOPの設定内容転送先データエリアのアドレスを取得。(sceNetcnfifGetAddr())
  3. 設定内容をIOPへ転送。(sceNetcnfifSendIOP())

その後、この関数を実行することによりIOP側のsceNetCnfEnv領域にsceNetcnfifDataの内容が設定されます。この関数は接続時、IOP側のsceNetCnfEnv領域を参照して接続を行うネットワークライブラリ用の関数です。あなたのネットワーク設定ファイル使用規定に基づき、接続時のリダイアル処理及び、接続機器の種別が矛盾する場合の接続処理を行っていますので使用は接続時に限ります。また、追加ATコマンドについて共通ネットワーク設定ライブラリで提供されるsceNetCnfConvA2S()関数の実行を行っています。（つまりIOP側のsceNetCnfEnv領域chat_additionalメンバにはチャットスクリプトが設定されます）この関数は実行する度にIOP側のsceNetCnfEnv領域を初期化します。そのため接続に必要な設定内容の設定は一度に行わなければなりません。接続時に組み合わせ、接続機器設定、接続プロバイダ設定の全てを必要とする場合は、全ての設定内容をIOPへ転送後、type == sceNetcnfifArg_type_netとして実行することにより、全ての設定内容がIOP側のsceNetCnfEnv領域に設定されます。また、組み合わせの内容を指定せずに接続機器設定と接続プロバイダ設定の両方を設定することはできません。接続機器設定のみ（type == sceNetcnfifArg_type_dev）、接続プロバイダ設定のみ（type == sceNetcnfifArg_type_ifc）の設定は可能です。
