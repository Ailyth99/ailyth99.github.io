1 ライブラリ概要

1. 1 概要

汎用ネットワークラッパー API(netglue) は、様々なネットワーク関連ライブラリをネットワークスタックライブラリとは独立に使用できるよう、汎用の下位ネットワークライブラリとして規定された BSD ソケットベースの API です。


netglue を下位 API として作成されたライブラリを使用する際には、使用するネットワークスタックライブラリに対応した netglue API ラッパーが必要となります。
SCEI スタック(libinet)用の netglue ライブラリは SDK に含まれていますので netglue ライブラリを作成する必要はなく、netglue API を意識する必要もありません。
netglue を利用したライブラリを、libinet 以外のネットワークスタックライブラリで使用する場合に、netglue ライブラリを作成する必要があります。

1. 2 関連ファイル

netglue ライブラリを作成するために必要なファイルは以下のとおりです。netglue 対応のネットワーク関連ライブラリをアプリケーションが利用する際に必要となるのは、各ネットワークスタックライブラリと、それ用のnetglue ライブラリのみです。

1. 3 libinsck 用 netglueライブラリのソース公開について

SCEI スタック(libinet)用の netglue ライブラリはソースを公開していますので、アプリケーション開発者が必要に応じて独自に修正しタイトルに組み込むことも可能です。

2. 2 BSD ソケット API

netglue API では基本的にほぼすべての関数が BSD ソケット API 準拠です。netglue の各 BSD ソケット API は、ブロッキングが前提です。ブロッキングを解除したい場合は、後述の netglue 独自 API であるsceNetGlueAbort() を他スレッドから呼び出します。

2. 3 netglue 独自 API

netglue では BSD ソケット API の他に、独自の API として下記の3つがあります。

sceNetGlueAbort()

指定した socket に対する他スレッドの各 BSD ソケット API のブロッキングを解除する。

sceNetGlueThreadInit()

netglue ライブラリを使用するに当たってスレッド毎に内部情報を保持するような初期化処理を行なう。netglue を使うスレッドはまず最初にこの API を呼ぶ。

sceNetGlueThreadTerminate()

sceNetGlueThreadInit() に対応する終了処理を行なう。netglue を使うスレッドは終了時にこの API を呼ぶ。

3 errno について

BSD ソケット API と同様、netglue では各 API のエラー値はsceNetGlueErrno で参照され、API としてはエラー時は単に負の値を返します。errno の各エラー値はライブラリリファレンスを参照して下さい。
