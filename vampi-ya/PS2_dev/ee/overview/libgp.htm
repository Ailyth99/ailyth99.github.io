1 ライブラリ概要

1. 1 機能概要

libgpはGSに対する描画処理を簡単に行えるようにするライブラリです。
GSプリミティブを描画する際には、適切なGSのレジスタに適切な順序で適切な値を設定するようにシーケンスを組み立ててGIFへ転送することになります。libgpでは描画機能の単位に相当する「パケット」を設けることによって、GSレジスタの操作シーケンスを意識する必要をなくしています。また、複数のパケットを「チェイン」として管理することにより、転送順序制御が容易に行えるようになっています。この転送順序制御機構はZソートやマテリアルソートなどにも応用できます。
このほかの特徴として、「特殊パケット」を用いて他のメモリデータやDMAチェインをまとめて転送できること、特にVU1へ転送するデータを同じチェインにまとめて転送できることが挙げられます。

1. 2 他のグラフィックスライブラリとの関連

libgpは上記のように、描画環境の初期設定と透視変換などの演算処理が終わった後の描画処理を担当するライブラリです。
描画環境の設定は、libgraphまたはHiG GSサービスを利用して行ってください。透視変換などの演算処理はlibvu0などを利用して独自に行うか、あるいは高レベルグラフィックスライブラリを利用すると演算から描画まで一貫して行うことができます。


参考

libgpはPlayStation用ライブラリでlibgpuが占めていたのと同様の位置付けにあります。ただし、libgpuに含まれていた機能のうち、libgraphに含まれるもの、描画エリアやディスプレイエリアの変更に関するもの、およびフォント関連の機能はlibgpには実装されていません。

2 libgpで使用するデータ構造

2. 1 チェイン管理構造体とオーダリングテーブル

チェインとは複数のパケットをひとまとめにして転送するための管理機構です。次のようなsceGpChain構造体として定義されていて、sceGpInitChain()を呼び出すことによって初期化されます。

typedef struct _sceGpChain {
    u_long128 *ot;	/* オーダリングテーブルのアドレス */
    u_long128 *pKick;	/* DMA 転送開始アドレス */
    u_long128 *pEnd;
    int resolution;
} sceGpChain;

*otはチェイン中のパケットの転送順序（描画順序）を示すオーダリングテーブルです。チェインを初期化する際に、描画順序を何段階で制御するかをresolutionによって指定しておき、パケットをチェインに追加する際にそのパケットの描画順序をlevelで指定します。転送の際には、levelの小さいところへ追加されたパケットから先に、またlevelの同じパケットどうしでは後から追加したものが先に転送されることになります。
また*otはDMAによって転送されるデータの実体でもあります。pKickはオーダリングテーブル中のDMA転送開始アドレスを、pEndはオーダリングテーブル中の最終DMA転送アドレスを保持しています。

2. 2 パケットタイプ

パケットには描画処理の内容に応じた多数のタイプがありますが、描画パケット / 環境パケット / 特殊パケットの3種に分けることができます。

  • 描画パケット：GSプリミティブに対応するパケット。
  • 環境パケット：アルファブレンディングやテクスチャの制御、またテクスチャやCLUT転送など、描画のための準備をするパケット。
  • 特殊パケット：他のDMAチェインなどをチェインに組み入れたりGSレジスタを直接制御するためのパケット。

パケットを生成するには、それぞれ適切なメモリを確保し、パケット初期化関数を用いて必要な値を設定します。このとき、引数でタイプを指定して汎用の初期化関数を呼び出す方法と、特定のタイプのパケットを初期化する専用関数を呼び出す方法とがあります。
汎用初期化関数でタイプを指定するためのシンボル定数が定義されていますので、このシンボル定数をパケットのタイプ名としても使用します。

2. 3 描画パケット

描画パケットはひとつひとつのGSプリミティブに対応するパケットです。64bit幅のRフォーマットと128bit幅のPフォーマットがあり、さらに、対応するGSプリミティブとテクスチャおよびシェーディングを表すサブタイプを加えて、描画パケットのタイプを指定します。
Rフォーマット（SCE_GP_PRIM_R）はGIFのREGLISTモードに対応する形式で、個々のデータが64bitで済みますが値を設定する際にビットフィールドの処理が必要になります。あらかじめデータを用意しておける場合や、処理時間よりもメモリ消費や転送時間を節約したい場合に有用です。一方のPフォーマット（SCE_GP_PRIM_P）はGIFのPACKEDモードに対応する形式です。個々のデータに128bit必要ですが、値を設定する処理は簡単になります。EE Core / VU0で計算した値を使用する場合や、メモリ消費や転送時間よりも処理時間を節約したい場合に有用です。

2. 5 特殊パケット

特殊パケットは、libgp本来の機能にないデータをチェインに組み入れるたり、GS レジスタを自由に設定するためのパケットです。次に示すパケットタイプが用意されています。

 パケットタイプ（構造体）   説明 
 SCE_GP_CALL 
 （sceGpCall） 		  callパケット。任意のDMAチェイン 
 （RETタグで終わるもの）を転送する 
 SCE_GP_REF 
 （sceGpRef） 		  refパケット。メモリ上の任意のデータを 
 転送する 
 SCE_GP_AD 
 （sceGpAd） 		  汎用レジスタ設定パケット。 
 任意のGS汎用レジスタに任意の値を設定する 


これらのパケットを使用して転送するDMAチェインやデータの内容は任意ですが、言うまでもなく、転送先で正しく扱われる内容・形式でなければなりません。

2. 6 パケットの構造

パケットの具体的な構造については、各タイプのパケットを表す構造体が定義されていますので、適宜リファレンスを参照してください。いずれも、GSに対する一連のコマンドシーケンスに適切なGIFタグを付けさらにDMAタグを付加して、そのままGIFへDMA転送できるデータ構造となっています。「EE User's Manual」のDMAの章、同じくGIFの章、また、「GS User's Manual」を参照すると、libgpの機能や動作についてよりよく理解できるでしょう。具体的なポイントをいくつか挙げておきます。

パケットは16バイトアライメントでなければならない

パケットはDMAで転送されるデータですので、16バイトバウンダリに配置されていなければなりません。オーダリングテーブルも同様で、メモリ確保の際に注意する必要があります。

パケットの内容を転送中に変更してはならない

転送開始後にパケットの内容を変更すると、変更前の内容と変更後の内容とどちらが転送されるか不定の結果となります。

予備領域を利用できることがある

描画パケットにあるuserregは、モノクロームパケットのパケットカラー設定領域として使用されますが、それ以外の場合（モノクロームでもパケットカラーを使用しない場合）には、任意のGSレジスタの設定に使用することができます。
たとえばtex0レジスタの設定に使用すると、SCE_GP_TEXENVパケットを省ける場合があります。

「テイル」について

libgp のパケットはすべて、次のパケットとの接続のための「テイル」を持っています。
テイルは、パケット構造体では DMAnext という名前のメンバで宣言されています。
後方登録の関数やパケットの削除のときにはこの「テイル」を引数として使用することがあります。

DMAタグの上位部分

DMAタグの上位64bitは構造体の定義としては空き領域になっていますが、libgpではパケットの先頭にあるDMAタグの上位部分を利用しています。パケットの先頭にあるDMAタグの3ワード目（bit95-64に相当する部分）にはVIFcode NOPがあり、IMMEDIATEフィールドをlibgp内部の管理用に使用しています。また、各DMAタグの4ワード目（bit127-96に相当する部分）には、PATH2経由で転送する場合のためのVIFcode DIRECTがあります。

3 描画処理の手順

libgpによる描画処理の全体的な流れは、まずチェインとパケットを準備し、チェインにパケットを登録し、チェインを転送する、という形になります。

3. 1 チェインの準備

チェインの準備は、sceGpChain構造体とオーダリングテーブルとに使用するメモリ領域をそれぞれ確保し、resolutionを指定してチェインを初期化する、という手順になります。プログラム例を次に示します。

sceGpChain chain;
u_long128 *ot;
ot = memalign(16, sceGpChkChainSize(resolution)*sizeof(u_long128));

sceGpInitChain(&chain, ot, resolution);
オーダリングテーブルはDMAで転送されるデータ実体ですので、16バイトアライメントでメモリを確保しなければなりません。サイズはresolutionによって変わりますので、上記のようにsceGpChkChainSize()を用いて求めてください。
resolutionは、そのチェインで描画順位を何段階に分けて制御するかという値です。実際的な利用方法としては、たとえば使用するテクスチャの個数をresolutionとしておき、後でパケットを登録する際には描画順位（level）としてテクスチャ番号を指定します。また、各テクスチャの環境設定とデータ転送パケットをそれぞれのlevelでチェインの最後に登録します。こうすると、テクスチャとそのテクスチャを使用するGSプリミティブとがまとめて転送されることになり、GSでの描画効率が向上します。もうひとつの利用例としては、Zソートを行うために使用する方法が挙げられます。画像の内容に応じて適当にZ値の分解能を定め、その値をresolutionとしてチェインを初期化しておきます。パケットはlevelの小さい順に転送されますから、描画パケットを登録する際にそれぞれのZ値に応じてlevelを指定すれば、Zソートが行われることになります。
なお、特に描画順序を制御する必要がない場合は、resolutionは1としてチェインを初期化し、すべてのパケットをlevel=0として登録すればよいでしょう。この場合、パケットの転送される順序は登録した逆順となります。

3. 2 パケットの準備

個々のパケットの準備は、それぞれ適切なメモリ領域を確保し、パケットタイプに応じて初期化し、パラメータをセットするという順序になります。

(1) メモリ領域の確保

描画パケットと汎用レジスタ設定パケットは、パケットの大きさが内容によって異なるため、メモリ領域の確保は次のようにして行います。

type = SCE_GP_PRIM_R | SCE_GP_TRISTRIP_GTU;	//パケットタイプ
arg = パケットのパラメータ;
ppacket = memalign(16, sceGpChkPacketSize(type, arg)*sizeof(u_long128));
sceGpInitPrimR(ppacket, type, arg);

環境パケットとcallパケット / refパケットは大きさが固定なので、それぞれパケットタイプに応じた構造体を用いてメモリ領域を確保することができます。

sceGpTexEnv packet;
sceGpInitPacket(&packet, SCE_GP_TEXENV, arg)

いずれにしても、パケットはDMA転送されるデータ実体ですから、16バイトアライメントでメモリを確保する必要があります。

(2) パケットの初期化

パケットの初期化は、汎用の初期化関数で引数としてパケットタイプを指定する方法と、パケットタイプごとの専用関数を使用する方法とがあります。
次のプログラム例は、先ほどの例で確保した描画パケットを汎用の初期化関数で初期化するものです。

sceGpInitPacket(ppacket, type, arg);

また次のプログラム例は、先ほどの例で確保したテクスチャ環境設定パケットを専用の初期化関数で初期化するものです。

sceGpInitTexEnv(&packet, ctxt);

(3) パラメータのセット

描画パケットについては、複数の頂点の座標やカラー値をパラメータとしてパケットにセットする必要があるため、手順がやや複雑になります。
まず、パラメータのインデックスを取得します。このためには次のように汎用のインデックス取得関数を使う方法と、パケットタイプごとの専用関数を使う方法があり、実行速度の点では専用関数が有利です。

/* TriangleStripパケットでi番目のカラー値 */
index = sceGpIndexRgba(type, i);	//汎用関数
index = sceGpIndexRgbaTriStripGTU(i);	//専用関数

次に、取得したインデックスを用いてパラメータの値をセットします。この方法も次のように、汎用のパラメータ設定関数を使う方法と、パケットタイプごとの専用関数を使う方法があります。

/* TriangleStripパケットへのカラー値の設定 */
sceGpSetRgba(ppacket, type, index, r, g, b, a);	//汎用関数
sceGpSetRgbaTriStripGTU(ppacket, index, r, g, b, a);	//専用関数

/* TriangleStripパケットへの座標値の設定 */
sceGpSetXyzf(ppacket, type, index, x, y, z, f);	//汎用関数
sceGpSetXyzfTriStripGTU(ppacket, index, x, y, z, f);	//専用関数

これらの関数はマクロとして実装され、型や値をチェックしていません。パケットタイプに合わない関数を呼び出したり、正しくない型にキャストされた状態で呼び出したり、間違ったindexを与えたりすると、パケットの内容やパケット外のメモリ領域を壊すことになりますので注意してください。
またこのほか、次のように値を直接構造体メンバにセットする方法もあります。

/* カラー値を構造体メンバのビットフィールドにセット */
ppacket->reg[index].rgba.R = r;
ppacket->reg[index].rgba.G = g;
ppacket->reg[index].rgba.B = b;
ppacket->reg[index].rgba.A = a;

/* u_long型のカラー値をそのままの形でセット */
u_long rgba[DATANUM];
ppacket->reg[index].ul= rgba[i];

環境パケットと特殊パケットについては、それぞれパケットタイプに応じた専用のパラメータ設定関数が用意されています。次に示すのは、テクスチャ環境設定パケットにパラメータをセットするプログラム例です。

sceGpSetTexEnv(&packet, &texarg, ctxt);

なお、テクスチャ関連パラメータをセットするためにテクスチャパラメータ構造体が定義されており、ほかにも Tim2 データを使用して設定する関数が用意されています。

3. 3 チェインへのパケットの登録

パケットの準備ができたら、次のようにしてチェインにパケットを登録します。

sceGpAddPacket(&chain, level, ppacket);

levelはそのパケットの描画順位を指定する値です。levelとして指定できる値の範囲は、0以上resolution未満の整数で、値の小さいlevelで登録したパケットから先に転送されます。ただし、「逆方向リセット」をおこなった場合に限り、大きいlevelで登録したパケットから先に転送されます。どちらの場合も、同じlevelのパケットどうしでは、後から登録したものが先に転送されます。
なお、チェインに登録するとパケット内のリンク情報が書き換えられるため、一度登録したパケットを再登録することはできません。同じ内容のパケットを繰り返し使用したい場合には、サブチェインを作成してsceGpCallChain()で登録するという方法があります。

3. 4 チェインへのパケットの登録その２・後方登録

もしも登録した通りの順序で転送をおこないたい場合には、次のようにしてパケットを登録します。

sceDmaTag *tail;
tail = sceGpGetTailChain(&chain, level);  /* 登録に使うテイルを取得 */
tail = sceGpAddPacket2(tail, ppacket0);
tail = sceGpAddPacket2(tail, ppacket1);
tail = sceGpAddPacket2(tail, ppacket2);

後方登録をおこなった場合は、後から登録したものは後に転送されます。上記の例では、ppacket0, ppacket1, ppacket2 の順に転送されることになります。同様にチェインを後方登録するための関数、 sceGpAddChain2(), sceGpCallChain2() も用意されています。

3. 5 転送処理

パケットを登録し終わったら、次のようにして転送処理を行います。

sceGpKickChain(&chain, path);

pathはチェインの転送先で、SCE_GP_PATH3（GIFへ）またはSCE_GP_PATH2（VIF1へ）を指定することができます。後者の場合、VU1へ送るデータをcallパケットなどを用いてチェイン中に組み入れておくことができます。
sceGpKickChain()はDMA転送を開始する関数で、転送終了を待たずにリターンしてきます。転送処理中にパケットの内容を変更すると不都合が起こる可能性がありますので、sceGsSyncPath()などを使用して転送終了を検出するようにしてください。転送が終了すれば、sceGpResetChain()を呼び出して、チェインを初期化直後の状態（パケットが登録されていない状態）に戻すことができます。

sceGsSyncPath(0, 0);	//転送終了待ち
sceGpResetChain(&chain);

3. 6 パケットを単独転送する

簡便のために、パケットを一つだけ、チェインを使用せずに単独で転送するための関数群が用意されています。この機能を使うには、次のように呼び出します。

sceGpInitPacket(ppacket, type, arg);	/* パケットの初期化 */

/* セット関数などでパケット内容を適宜設定する */

sceGpPacketMode(ppacket, 1);		/* パケットを単独転送モードにする */
sceGpSyncPacket(ppacket);		/* パケット領域のキャッシュを掃き出す */
sceGpKickPacket(ppacket, path);		/* パケットを単独で DMA 転送 */
単独転送モードは、テイルの DMA タグを END タグに切り替えることによって実装されています。

3. 7 描画パケットの頂点座標値レジスタ(XYZF2 / XYZ2)の指定

描画パケット中の頂点座標値レジスタを切り替えることが出来ます。XYZF2 / XYZ2 が選択可能で初期のデフォルトはXYZF2です。デフォルトを切り替えると、その後のInit関数で設定される値が変わります。

sceGpSetDefaultZ32(1);			/* デフォルト値の変更 */
/* 移行の init 関数では出力レジスタが XYZ2 となる */

/* デフォルトを変更せずに個別に変更するばあい */
sceGpInitPacket(ppacket, SCE_GP_PRIM_P | SCE_GP_TRI_F, 1);  /* 描画パケット初期化 */
sceGpSetZ32(ppacket, 1);				  /* 出力レジスタ変更 */
sceGpSetXyz32(ppacket, sceGpIndexXyzfTriF(0), x0, y0, z0); /* 頂点座標値のセット */
