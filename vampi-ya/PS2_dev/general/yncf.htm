1 このドキュメントについて

このドキュメントでは、ネットワーク接続に関する情報を保存するファイルとして弊社が定めている「あなたのネットワーク設定ファイル」の内容等と、あなたのネットワーク設定ファイルを使用するプログラムの作成基準について解説したものです。

1. 1 バージョン管理について

このドキュメントは、TRCと合わせてプログラミング上の規定事項となるため、単独にバージョン管理されています。
バージョンの管理方法は、以下のとおりです：

作成基準の項目追加、削除のあった場合……一の位を変更
例) Version 1.0 → Version 2.0

誤記修正、内容の補足のあった場合……小数点第一位を変更
例) Version 1.0 → Version 1.1

2 あなたのネットワーク設定ファイル

2. 1 あなたのネットワーク設定ファイルとは

あなたのネットワーク設定ファイルとは、ネットワーク接続に使用する機器や接続先プロバイダのアクセスポイントやユーザーアカウントなどユーザーごとの個別情報を、ユーザーが一度入力・設定すれば各アプリケーションで共通して利用できるようにするため規定されたネットワーク設定ファイルです。
ファイル形式は共通ネットワーク設定ライブラリ（netcnf）で規定されている共通ネットワーク設定ファイルのサブセットですが、一般的なネットワークアプリケーションで必要とする設定項目をカバーし、また、保存場所などの運用方法を規定して、より容易にアプリケーション間で共有できるようになっています。



あなたのネットワーク設定ファイルに対する操作は、共通ネットワーク設定ライブラリ（netcnf）及び、共通ネットワーク設定インタフェースライブラリ(netcnfif)を通して行わなければなりません。また、あなたのネットワーク設定ファイルは共有のリソースですので、これを使用するアプリケーション（以下ネットワーク設定アプリケーションと呼びます）は一定の基準に従う必要があります。

2. 2 あなたのネットワーク設定ファイルの構成

あなたのネットワーク設定ファイルは以下のファイルによって構成されます。
  • 設定管理ファイル
  • 組み合わせファイル
  • 接続機器設定ファイル
  • 接続プロバイダ設定ファイル
これらのファイルに加えて保存先の各デバイスに応じて必要なファイルが追加されます。保存先のデバイスとしてはメモリーカード(PS2)とハードディスクドライブがあり、各デバイスでの保存場所とファイル構成は以下のようになっています。

2. 3 あなたのネットワーク設定ファイルの内容

あなたのネットワーク設定ファイルはファイルごとに保存可能な設定内容が定められ、設定内容に応じて保存しなければならない設定項目が決められています。ファイルごとに保存可能な設定内容と設定内容ごとに保存しなければならない設定項目を以下に示します。
なお、設定項目の表における区分欄の表示は次の意味を持ちます。

 区分   意味 
 必須   ユーザーが必ず入力・選択し、設定されなければならない設定項目 
 必須・自動   ネットワーク設定アプリケーションが内部で判断し、 
 自動的に設定されなければならない設定項目

また、該当キーワード欄は、その設定項目に対応する共通ネットワーク設定ライブラリ(netcnf)で定められたキーワードを示しています（キーワードの詳細は共通ネットワーク設定ライブラリ(netcnf)概要を参照してください）。

3 ネットワーク設定アプリケーションの作成基準（読み出し処理）

あなたのネットワーク設定ファイルを読み出して使用するアプリケーションでは、それぞれの処理を以下で解説する基準に従って実装する必要があります。

3. 1 組み合わせの表示・選択

組み合わせを表示・選択する処理を実装する場合は、以下の基準に従ってください。
  • ユーザーが組み合わせを選択した場合、選択された組み合わせに登録されている接続機器設定と接続プロバイダ設定が表示されること。また、接続機器設定あるいは接続プロバイダ設定のいずれかまたは両方が登録されていない（削除された）組み合わせを選択可能な場合、それぞれの設定が存在しないため使用不可である旨のメッセージが表示されること。
  • メモリーカード(PS2)に保存された組み合わせを表示する場合、設定名（組み合わせ名）は次のいずれかの形式で表示されること。
    • 「組み合わせ1」〜「組み合わせ6」
    • 「Network configuration1」〜「Network configuration6」
    • 「Combination1」〜「Combination6」
    • 「1」〜「6」
  • ハードディスクドライブに保存された組み合わせを表示する場合、設定名（組み合わせ名）は次のいずれかの形式で表示されること。
    • 「組み合わせ1」〜「組み合わせ10」
    • 「Network configuration1」〜「Network configuration10」
    • 「Combination1」〜「Combination10」
    • 「1」〜「10」
  • 「あなたのネットワーク設定ファイルの構成」で定められている最大ファイル数の組み合わせをすべて表示・選択できることが望ましい。
  • デフォルトデータとして独自のデータ内に保存してある組み合わせを表示する場合、元となるあなたのネットワーク設定ファイルディレクトリ内に同じ内容の組み合わせが存在することを確認すること。この時、組み合わせに登録されている接続機器設定と接続プロバイダ設定の存在およびその内容も確認すること。

3. 2 組み合わせを用いた接続

組み合わせを用いてネットワークへの接続を行う処理を実装する場合は、以下の基準に従ってください。
  • 以下の条件が成り立つ組み合わせをユーザーが選択できないようになっていること、または、その組み合わせを選択した場合に接続できないことが表示されること。
    1. 登録されている接続機器設定に記述されている機器が物理的に接続されていない
    2. 組み合わせに登録されている接続機器設定と接続プロバイダ設定のいずれか、または両方が存在しない
  • デフォルトデータとして独自のデータ内に保存してある組み合わせを使用して接続する場合、元となるあなたのネットワーク設定ファイルディレクトリ内に同じ内容の組み合わせが存在することを確認すること。また、組み合わせに登録されている接続機器設定と接続プロバイダ設定の存在およびその内容も確認すること。
  • 接続先電話番号１、接続先電話番号２、接続先電話番号３が設定されている場合、小さい番号の電話番号が話し中の場合、大きい電話番号へリダイアルする仕組みになっていること。
  • イーサネットアダプタの種別について、以下のように接続プロバイダ設定が矛盾している場合でも接続できるようになっていること。
    1. SCE製ネットワークアダプタのイーサネットアダプタで接続しようとする場合に、接続機器設定はtype nicだが接続プロバイダ設定がtype ethとなっている
    2. SCE製以外のネットワークアダプタのイーサネットアダプタで接続しようとする場合に、接続機器設定はtype ethだが接続プロバイダ設定がtype nicとなっている

3. 7 あなたのネットワーク設定ファイルの読み込み時のエラーについて

  • sceNETCNF_MAGIC_ERRORを検出した場合には、他の"PlayStation 2"で作成されたあなたのネットワーク設定ファイルであり使用できない旨のメッセージを出力し、あなたのネットワーク設定ファイルを削除するかどうかを尋ねる旨のメッセージを出力し、削除しない場合はそのあなたのネットワーク設定ファイルを無視すること。

4 ネットワーク設定アプリケーションの作成基準（書き込み処理）

あなたのネットワーク設定ファイルを操作（追加・編集・削除）する機能を持つアプリケーションでは、それぞれの処理を以下で解説する基準に従って作成する必要があります。読み出し処理については第3章で解説してある作成基準に従って作成してください。
この章で解説する機能を持たず、他のアプリケーションで作成されたあなたのネットワーク設定ファイルを使用するのみのアプリケーションに関しては、この章の規定は適用されません。

4. 1 組み合わせの追加・編集・削除

組み合わせを操作（追加・編集・削除）する処理を実装する場合は、以下の基準に従ってください。
  • 共通ネットワーク設定ライブラリ(netcnf)を使用すること。
  • 組み合わせを追加・削除できること（既存の組み合わせを編集するだけという仕様は不可）。
  • 定められたディレクトリに正しいファイル名で保存されること（「あなたのネットワーク設定ファイルの構成」を参照）。
  • 定められた最大ファイル数より多い数の組み合わせファイルを保存できないこと（「あなたのネットワーク設定ファイルの構成」を参照）。
  • 定められたすべての設定項目に対応していること。同時に、定められた設定項目以外の設定項目を保存しないこと（「あなたのネットワーク設定ファイルの内容」を参照）。
  • 別のデバイスにファイルとして存在する接続機器設定と接続プロバイダ設定とを組み合わせに登録しないこと。
  • メモリーカード(PS2)に保存する場合、設定名（組み合わせ名）は以下のいずれかの形式とすること。
    • 「組み合わせ1」〜「組み合わせ6」
    • 「Network configuration1」〜「Network configuration6」
    • 「Combination1」〜「Combination6」
    • 「1」〜「6」
  • ハードディスクドライブに保存する場合、設定名（組み合わせ名）は以下のいずれかの形式とすること。
    • 「組み合わせ1」〜「組み合わせ10」
    • 「Network configuration1」〜「Network configuration10」
    • 「Combination1」〜「Combination10」
    • 「1」〜「10」
  • 編集時、未知の設定項目があった場合にはその設定項目は変更せずに保存すること。
  • デフォルトデータとする以外の目的で、アプリケーション独自のデータ内に同じ内容を保存しないこと。

4. 2 接続機器設定の追加・編集・削除

  • 共通ネットワーク設定ライブラリ(netcnf)を使用すること。
  • 使用する接続機器設定を追加・削除できること（既存の接続機器設定を編集するだけという仕様は不可）。

  • 定められた最大ファイル数より多い数の接続機器設定ファイルを保存できないこと（「あなたのネットワーク設定ファイルの構成」を参照）。

  • 編集時、未知の設定項目があった場合にはその設定項目は変更せずに保存すること。

  • 追加ATコマンドとしては、共通ネットワーク設定ライブラリで提供されているsceNetCnfConvA2S()関数で変換可能なATコマンド、および、共通ネットワーク設定ライブラリ概要（overview）で定義されているチャットスクリプトを入力可能とすること。入力された文字列に対してsceNetCnfConvA2S()関数を実行してから保存すること。その際にsceNETCNF_TOO_LONG_STR(-19)が検出された場合、入力文字数が多い旨のメッセージを出力すること。
  • デフォルトデータとする以外の目的でアプリケーション独自のデータ内に同じ内容を保存しないこと。

4. 3 接続プロバイダ設定の追加・編集・削除

接続プロバイダ設定を操作（追加・編集・削除）する処理を実装する場合は、以下の基準に従ってください。

  • 接続プロバイダ設定を追加・削除できること（既存の接続プロバイダ設定を編集するだけという仕様は不可）。

4. 4 メモリーカード(PS2)への保存

あなたのネットワーク設定ファイルをメモリーカード(PS2)に保存する処理を実装する場合は、以下の基準に従ってください。
  • 共通ネットワーク設定ライブラリ(netcnf)を使用すること。
  • 定められたディレクトリに正しいファイル名で保存すること（「あなたのネットワーク設定ファイルの構成」を参照）。
  • 接続機器設定ファイル、接続プロバイダ設定ファイルは符号化して保存されていること。
  • 定められた最大ファイル数より多い数の設定ファイルを保存できないこと（「あなたのネットワーク設定ファイルの構成」を参照）。最大数を超えて保存しようとした場合には、その設定が最大数に達している旨のメッセージを出力すること。
  • 94Kbytes以上の空き容量がない場合、保存時に空き容量がない旨のメッセージを出力すること。なお、あなたのネットワーク設定ファイルを削除する際にも、設定管理ファイルが保存されるため、空き容量のチェックが必要となる。
  • アイコンファイルとicon.sysファイルは指定のものを使用すること。
  • その他、メモリーカード(PS2)に関するTRCに準拠していること。

4. 5 ハードディスクドライブへの保存

あなたのネットワーク設定ファイルをハードディスクドライブに保存する処理を実装する場合は、以下の基準に従ってください。
  • 共通ネットワーク設定ライブラリ(netcnf)を使用すること。
  • 定められたディレクトリに正しいファイル名で保存すること（「あなたのネットワーク設定ファイルの構成」を参照）。
  • 接続機器設定ファイル、接続プロバイダ設定ファイルは符号化して保存されていること。
  • 定められた最大ファイル数より多い数の設定ファイルを保存できないこと（「あなたのネットワーク設定ファイルの構成」を参照）。最大数を超えて保存をしようとした場合には、その設定が最大数に達している旨のメッセージを出力すること。
  • 244Kbytes以上の空き容量がない場合、保存時に空き容量がない旨のメッセージを出力すること。なお、あなたのネットワーク設定ファイルを削除する際にも、設定管理ファイルが保存されるため、空き容量のチェックが必要となる。
  • その他、ハードディスクドライブに関するTRCに準拠していること。

5 実装上の詳細基準

5. 1 接続時のリダイアル処理

ネットワーク設定アプリケーションで接続を行う際に、あなたのネットワーク設定ファイルで接続先電話番号1〜3が設定されている場合は、接続先電話番号１が話し中であれば接続先電話番号２に、さらに話し中であれば接続先電話番号３に、というリダイアル処理が必要です（リダイアルの判定は各モデムドライバ付属のダイアリング定義ファイル（DIAL_CNF）で定義されたredial_string項目に従って行われます）。sceNetCnfLoadEntry()を用いてあなたのネットワーク設定ファイルを読み込んだだけではリダイアルの設定が不十分ですので、必ず以下の設定を行ってください。
  • 接続時において接続先電話番号１〜３が設定されていた場合、設定されている接続先電話番号の個数から１を引いた値をsceNetCnfInterface構造体のメンバredial_countにセットする。
実装例は以下のようになります。

sceNetCnfEnv_t env;
int i, redial_count;

// sceNetCnfLoadEntry() を実行
sceNetCnfLoadEntry("設定管理ファイル名", type, "設定名(組み合わせ名)", &env);

// redial_count を設定する
for(i = 0, redial_count = 0; i < sceNetCnf_MAX_PHONE_NUMBERS; i++){
	if(NULL == (env.root->pair_head->ifc->phone_numbers[i])) continue;
	switch(i){
	case 0: redial_count++; break;
	case 1: redial_count++; break;
	case 2: redial_count++; break;
	}
}
env.root->pair_head->ifc->redial_count = redial_count - 1;
なお、上記処理により設定したredial_count値をあなたのネットワーク設定ファイルとして保存することは禁止します。接続時のみの処理として実装してください。

5. 2 接続機器の種別が矛盾する場合の接続処理

共通ネットワーク設定ライブラリ（netcnf.irx）の仕様では接続プロバイダ設定で指定されている接続機器の種別が有効になるため、たとえば以下のような場合に接続不可となります。
  1. 接続プロバイダ設定がtype eth、接続機器設定がtype nicでSCE製ネットワークアダプタのイーサネットアダプタを用いて接続しようとする場合
  2. 接続プロバイダ設定がtype nic、接続機器設定がtype ethでSCE製ネットワークアダプタ以外のイーサネットアダプタを用いて接続しようとする場合
ネットワーク設定アプリケーションで接続を行う際には、このような場合でも接続できることが必要ですので、以下のような実装を行ってください。
  • PPPoEでなく接続プロバイダ設定のtypeがpppでない場合は、接続プロバイダ設定のtypeを接続機器設定のtypeで上書きする。
実装例は以下のようになります。

sceNetCnfEnv_t env;

// sceNetCnfLoadEntry() を実行
sceNetCnfLoadEntry("設定管理ファイル名", type, "設定名(組み合わせ名)", &env);

// PPPoE でなく接続プロバイダ設定の type が
// ppp でない場合は type 合わせをする。
if(env.root->pair_head->ifc->pppoe != 1 &&
  env.root->pair_head->ifc->type != sceNetCnf_IFC_TYPE_PPP){
	env.root->pair_head->ifc->type = env.root->pair_head->dev->type;
}
なお、上記処理により設定した値をあなたのネットワーク設定ファイルとして保存することは禁止します。接続時のみの処理として実装してください。
