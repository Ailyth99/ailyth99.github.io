1 概要

このドキュメントでは、DBGPのEEスレッド拡張について説明します。これらの機能は、DBGP Version 3.10以降でサポートされています。サポートされていないバージョンでは、DBGPヘッダresultフィールドにDBGP_RESULT_INVALREQを返します。

1. 1 メッセージフォーマット

DBGPヘッダの各フィールドに設定する値は次のとおりです。

id

DBGP_CPUID_CPU　0　// CPU (ESDBGP)

group

DBGP_GROUP_EE_THREAD　1

type

 DBGP_EE_THREAD_TYPE_THREADLIST   0x30   //Get Thread List Request 
 DBGP_EE_THREAD_TYPE_THREADLISTR   0x31   //Get Thread List Reply  
 DBGP_EE_THREAD_TYPE_TCB   0x32   //Get TCB Request 
 DBGP_EE_THREAD_TYPE_TCBR   0x33   //Get TCB Reply 
 DBGP_EE_THREAD_TYPE_SEMABLOCK   0x34   //Get Semaphore Block Request 
 DBGP_EE_THREAD_TYPE_ SEMABLOCKR   0x35   //Get Semaphore Block Reply 
 DBGP_EE_THREAD_TYPE_HANDLERLIST   0x36   //Get Handler List Request 
 DBGP_EE_THREAD_TYPE_HANDLERLISTR   0x37   //Get Handler List Reply 
code、countは、typeの説明で説明します。

result

ターゲットからホストへのメッセージにおいて、次の表に示す結果コードが入ります。ホストからターゲットへ送信するメッセージでは常に 0 を設定します。
全グループ共通
 DBGP_RESULT_GOOD   0x00   // Good 
 DBGP_RESULT_INVALREQ   0x01   // Invalid Request 
 DBGP_RESULT_UNIMPREQ   0x02   // Unimplemented Request 
 DBGP_RESULT_ERROR   0x03   // Error 

ターゲットがタイプを認識した後に不正要求であると判断した場合や、認識できるが実装していないタイプのメッセージを受けとった場合は、それぞれ適切なタイプにはその応答タイプのメッセージを返します。

なお、このドキュメントで明示されていないフィールドには、送信側は常に 0 を設定しなければなりません。また受信側は、0 でない値を得たときにはできるだけエラー扱いとしなければなりません。

2 メッセージ

2. 1 Thread List(THREADLIST/THREADLISTR)

EEカーネル上の有効なスレッドリストの要求メッセージとその応答メッセージです

以下のようなヘッダとデータで構成されます。

 THREADLIST 		  DECI2 ヘッダ + DBGP ヘッダ 
 DBGP_EE_THREAD_TYPE_ 
 THREADLISTR 		  DECI2 ヘッダ + DBGP ヘッダ +  
 DBGP_EE_THREADLIST_HDR +  
 DBGP_EE_THREADLIST_DATA ×  
 DBGP_EE_THREADLIST_HDR.count 

DBGPヘッダのcodeフィールドには、

 DBGP_EE_THREAD_CODE_ 
 ALLTHREADLIST 		  0x3f   全スレッドのリスト 
 DBGP_EE_THREAD_CODE_ 
 THREADREADYQUEUELIST 		  0x0   スレッドレディキューの 
 スレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 RUN 		  0x1   RUN状態のスレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 READY 		  0x2   READY状態のスレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 WAIT 		  0x4   WAIT状態のスレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 SUSPEND 		  0x8   SUSPEND状態のスレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 WAITSUSPEND 		  0x20   WAITSUSPEND状態の 
 スレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 DORMANT 		  0x10   DORMANT状態のスレッドリスト 
 DBGP_EE_THREAD_CODE_ 
 RESET 		  0x80   スレッドがRUN状態になった 
 回数を0に初期化します。

2. 3 Semaphore Block(SEMABLOCK/SEMABLOCKR)

EEカーネル上の有効なセマフォ情報を要求するメッセージと、セマフォ情報を返す応答メッセージです。以下のようなヘッダとデータで構成されます。

 DBGP_EE_THREAD_TYPE_ 
 SEMABLOCK 		  DECI2 ヘッダ + DBGP ヘッダ + 
  DBPG_EE_SEMAID_DATA 
 DBGP_EE_THREAD_TYPE_ 
 SEMABLOCK R 		   DECI2 ヘッダ + DBGP ヘッダ +  
 DBGP_EE_SEMABLOCKHDR +  
 (DBGP_EE_SEMABLOCK_DATA +  
 DBGP_EE_THREADID_DATA × 
 DBGP_SEMABLOCK_DATA.numWaitThreads )× 
 DBGP_EE_THREADBLOCK_HDR.count 
DBGP_EE_THREAD_TYPE_SEMABLOCKのDBGPヘッダのcodeフィールドには、
DBGP_EE_THREAD_CODE_SEMAID 0 //セマフォIDをDBGP_EE_SEMAID_DATAに指定してセマフォ情報を取得します。
DBGP_EE_THREAD_CODE_SEMAALL　1　//すべてのセマフォ情報を取得します。
DBGP_EE_SEMAID_DATAは、0にしてください。

図 5 セマフォIDデータ(DBGP_EE_SEMAID_DATA)


sema id

セマフォID

図 6 セマフォブロックデータ(DBGP_EE_SEMABLOCK_HDR)


count

応答メッセージに含まれるセマフォの数

図 7 セマフォの情報(DBGP_EE_SEMABLOCK_DATA + DBGP_EE_THREADID_DATA×DBGP_EE_SEMABLOCK_DATA.numWaitThreads)


sema id

セマフォID

count

セマフォ資源数

maxcount

セマフォ資源の最大数

attr

セマフォ属性

option

ユーザー定義の付加情報

numWaitThreads

セマフォ待ちをしているスレッドの数

thread id

セマフォ待ちをしているスレッドのスレッドID。このIDはセマフォ待ちに入った順番で返される。

DBGP_EE_THREAD_TYPE_SEMABLOCKR応答のDBGPヘッダのcountフィールドには、該当セマフォが存在するとき1を返します。存在しないとき0を返します。

2. 4 Handler List(HANDLERLIST/HANDLERLISTR)

EEカーネルAPIを用いて登録された割込みハンドラのリストの要求メッセージとその応答メッセージです。以下のようなヘッダとデータで構成されます。
 DBGP_EE_THREAD_TYPE_ 
 HANDLERLIST 		   
 DBGP_EE_THREAD_TYPE_ 
 HANDLERLISTR 		  DECI2ヘッダ + DBGPヘッダ + 
 DBGP_EE_HANDLER_DATA ×DBPGヘッダ.count 

図 8 割り込みハンドラデータ(DBGP_EE_HANDLER_DATA)


kind

ハンドラの種類
DBGP_EE_INTC_HANDLER 0 // INTC割り込み
DBGP_EE_DMAC_HANDLER 1 // DMAC割り込み

number

割り込みハンドラ番号(0-15)

mask

割り込みハンドラが有効かどうか

図 9 DBGP_EE_HANDLER_DATA.mask


ハンドラが有効な時には、該当番号のビットが1になります。無効時は、0です。

func

割り込みハンドラ関数のエントリーポイント

arg

割り込みハンドラへの引数

DBGP_EE_THREAD_TYPE_HANDLERLISTR応答は、DBGPヘッダのcountフィールドに返すDBGP_EE_HANDLER_DATAの個数を返します。
