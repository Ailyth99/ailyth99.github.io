typedef struct {
	u_char stat;
	u_char second;
	u_char minute;
	u_char hour;
	u_char pad;
	u_char day;
	u_char month;
	u_char year;

メンバ

 stat   0:正常 それ以外はエラー（内蔵電池切れ等） 
 second   秒（BCD値） 
 minute   分（BCD値） 
 hour   時（BCD値） 
 pad   アライメントによって生じるパディングデータ 
 day   日（BCD値） 
 month   月（BCD値） 
 year   年（BCD値） 

解説

sceCdBreak
コマンドの中断
形式

int sceCdBreak ( void )
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能
マルチスレッドセーフである（割り込み禁止、許可状態に依存しない）

引数

なし

返り値

成功の場合、1を返します。
エラーの場合、0を返します。sceCdGetError()によって以下のエラー内容を取得できます。
 値   マクロ   結果 
 0x01   SCECdErABRT   アボートコマンドを受け取った 


解説

実行中のコマンド（sceCdPause(), sceCdRead(), sceCdSeek(), sceCdStandby(), sceCdSstatus(), sceCdstop()等）を中断します。中断処理の終了の確認はsceCdSync()関数によって行います。
各コマンドの処理は中断され、コールバック等が設定してある場合はコールバック関数が呼び出されます。

sceCdCallback
sceCdSyncコールバック関数を定義する
形式

int sceCdCallback (
	void (*func)(int)
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである（割り込み禁止、許可状態に依存しない）

引数

 func   コールバック関数のアドレス 

返り値

前回設定したコールバック関数のアドレスを返します。
未設定の場合、０を返します。

解説

ノンブロック関数の終了時に呼ばれるコールバックfuncを設定します。
コールバックを設定するとノンブロック関数終了時に関数funcが呼び出されます。
関数funcは割り込みハンドラで呼び出されます。
funcに０を指定した場合およびコマンド発行に失敗した場合はコールバックは発生しません。
また、すでにコールバックの要因となる関数が実行中の場合もコールバックの設定は行われません。
コールバック関数の第一引数には、コールバックの要因となった関数のコードを渡します。
 マクロ   結果 
 SCECdFuncRead   sceCdRead()関数が終了した 
 SCECdFuncSeek   sceCdSeek()関数が終了した 
 SCECdFuncStandby   sceCdStandby()関数が終了した 
 SCECdFuncStop   sceCdStop()関数が終了した 
 SCECdFuncPause   sceCdPause()関数が終了した 

コールバック関数内での注意事項

  1. コールバック関数は割り込みハンドラ中に割り込み禁止状態で呼ばれるため、できる限り早く処理を終える必要があります。また、ライブラリにて割り込み処理専用の関数が用意されている場合、その関数を使用する必要があります。

sceCdChangeThreadPriority
EE側リクエスト処理モジュールのIOPスレッドプライオリティ変更
形式

int sceCdChangeThreadPriority(
	int priority
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフでない（割り込み禁止、許可状態に依存しない）

引数

 priority   EE側リクエスト処理モジュールの 
 IOPスレッドプライオリティ値 

返り値

成功の場合、0を返します。
エラーの場合、iop側KernelErrorCodeの値を返します。

解説

EE側リクエスト処理モジュールのIOPスレッドプライオリティ値を設定します。
デフォルトのEE側リクエスト処理モジュールのIOPスレッドプライオリティ値は81です。
またIOPスレッドプライオリティ値を変更する場合、他のモジュールのプライオリティに対して細心の注意を払う必要があります。そのため、安易にIOPスレッドプライオリティ値を変更する事はお勧めしません。

sceCdDiskReady
ドライブの状態をチェックする
形式

int sceCdDiskReady (
	int mode
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである（割り込み許可状態で呼ぶこと）

引数

 mode   チェックモード（0: ブロック型、1: ノンブロック型） 
 ノンブロック型の使用に際しては他スレッド動作状況を 
 十分考慮する必要がある。 

返り値

 値   マクロ   結果 
 0x02   SCECdComplete   コマンドを発行できる状態にある 
 0x06   SCECdNotReady   ドライブはコマンドを受け付けられない 


解説

ドライブの状態をチェックし、コマンドが発行できるかどうかを調べます。
ドライブ内のメディア種が判定中の場合はSCECdNotReadyとなります。
ブロック型の場合は、ドライブの回転が安定していなければ安定するまでホールドしてリターンします。ノンブロック型の場合は、ステータスを調べてただちにリターンします。
本関数をマルチスレッドの環境下でノンブロック型でポーリングに使用する場合は必ず、DelayThread()関数等を使用して、他のスレッドが動作する余地があるように考慮する必要があります。
