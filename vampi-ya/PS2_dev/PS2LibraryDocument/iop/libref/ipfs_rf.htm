format
ファイルシステムのフォーマット
形式

#include <sys/mount.h>
int format(
	const char *devname,
	const char *blockdevname,
	void *arg,
	int arglen
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 devname   ファイルシステムのデバイス名(pfs:) 
 blockdevname   あらかじめ作成されているパーティションの 
 ブロックデバイス名 
 (例:"hdd0:BISLPS-XXXXXX,fpwd") 
 arg   ゾーンサイズ変数とフラグメントオプションへのポインタ 
 arglen   argのサイズ。 

返り値

成功の場合、0を返します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -2   -ENOENT   指定されたパーティションが存在しない 
 -5   -EIO   入出力エラー 
 -6   -ENXIO   サポートされているデバイスではな 
 -12   -ENOMEM   十分な空きメモリが無い 
 -13   -EACCES   アクセス権限がない 
 -16   -EBUSY   すでに指定されたパーティションが 
 オープンされている 
 -19   -ENODEV   指定されたデバイスが存在しない 
 -22   -EINVAL   不正な引数が指定された 
 -24   -EMFILE   オープンできるディスクリプタの 
 最大数に達している 


解説

新たにファイルシステムを構築します。ゾーンサイズに指定できる値は、2K - 128K の範囲の2のn乗の数のみです。ゾーンサイズは、作成予定のファイルの大部分が小さければ小さく、大きければ大きく設定した方が効率がよくなります。

例:
int zonesz = 8192;
format("pfs:", "hdd0:BISLPS-XXXXXX,fpwd", &zonesz, sizeof(int));

また開発時の検証用に故意にフラグメンテーションを発生させた状態でフォーマットすることができます。

int arg[3];
arg[0] = 8192;              // zone size
arg[1] = 0x00002d66;        // -f
arg[2] = 0x01030f0f;        // fragment bit pattern
sceFormat("pfs:", "hdd0:test", &arg, sizeof(arg));

ビットパターンの 1 ビットが、1 ゾーンに対応します。例えば、0x0f0f0f0f を指定すると 4 ゾーンが使用中、4 ゾーンが空きといったパターンの繰り返し状態でフォーマットされます。

備考

この操作を行うと既に指定されたパーティション上に作成されていたファイルシステムは初期化されてしまいますので、ご注意ください。

lseek
ファイルポインタの移動
形式

#include <stdio.h>
int lseek(
	int fd,
	long offset,
	int whence
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 fd   ファイルディスクリプタ 
 offset   ポインタの移動距離 
 whence   パーティションの拡張アトリビュート領域内のoffsetの基準位置。 
 以下のいずれかの定数で指定する。 
     SEEK_SET   開始位置 
     SEEK_CUR   現在位置 
     SEEK_END   最後 

返り値

成功の場合、新たに設定されたファイルポインタの値を返します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -5   -EIO   入出力エラー 
 -9   -EBADF   fdが有効なオープンされたディスクリプタでない 
 -21   -EISDIR   ディレクトリに対して要求した 
 -22   -EINVAL   whenceが正しい値ではない。 
 もしくは、EOFをこえるoffsetが指定された。 
 -27   EFBIG   移動量が、符号なし 32bit の整数の範囲を超えた 
 (この場合でも、移動操作は正しく行われる) 


解説

ファイルディスクリタfdのオフセットをwhenceに従ってoffsetの位置へ変更します。オフセットをファイルの最後を超えた場所に設定することはできません。

mount("pfs2:", "hdd0:tst3,,rpwd3", SCE_MT_RDONLY, NULL, 0);
