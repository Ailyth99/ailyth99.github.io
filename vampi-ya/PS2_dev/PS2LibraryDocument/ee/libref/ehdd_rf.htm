 以下のいずれかの定数で指定する。 
    SCE_SEEK_SET   開始位置 
    SCE_SEEK_CUR   現在位置 
    SCE_SEEK_END   最後

sceOpen
メインパーティションの作成、オープン
形式

#include <sifdev.h>
int sceOpen(
	const char *name,
	int flags
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 name   パーティション識別文字列。 
 flags   アクセスモード。次のいずれかの定数で指定します。 
 複数指定する場合は論理和をとる。 
    SCE_RDONLY   読み出し専用にオープンする 
    SCE_RDWR   読み書き可能でオープンする 
    SCE_CREAT   パーティションが存在しなければ新規作成する 

返り値

正常終了の場合、ファイルディスクリプタ(0以上の値)を返します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -2   -ENOENT   指定されたパーティションが存在しない 
 -5   -EIO   入出力エラー 
 -12   -ENOMEM   十分な空きメモリが無い 
 -13   -EACCES   アクセス権限がない 
 -16   -EBUSY   すでに指定されたパーティションが 
 オープンされている 
 -19   -ENODEV   指定されたデバイスが存在しない 
 -22   -EINVAL   不正な引数が指定された 
 -24   -EMFILE   オープンできるディスクリプタの最大数に 
 達している 
 -28   -ENOSPC   空き領域が存在しない 


解説

メインパーティションの作成、オープンを行います。オープンしたファイルに対してはファイルディスクリプタが与えられます。あるパーティションを同時にオープンすることはできません。パーティション識別文字列は、デバイス名 + ユニット番号 + ':' + カンマで区切った以下の順の文字列となります。

 文字列   説明 
 パーティションID   ディスク全体でユニークな ID で正式には SCE から 
 発行しますが、作成段階ではユニークな文字列であれ 
 ば何でも構いません。長さは32文字までです。 
 フルパスワード   リードライトアクセスする際に必要なパスワード、 
 8文字まで。 
 読み取り専用 
 パスワード 		  リードアクセスのみを行う場合に必要なパスワード、 
 8文字まで。 
 パーティション 
 サイズ 		  パーティションのサイズを文字列で指定します。 
 指定可能な文字列は、以下のとおりです。 
 128M、256M、512M、1G、2G、4G、8G、16G、32G 
 ファイルシステム名   現状では、PFS のみ有効です。 


作成の場合は、パスワード以外はすべて指定する必要があります。オープンする場合は、パーティションをオープンするために必要なパスワードまでを指定します。

例1: パスワードを指定して作成する場合
sceOpen("hdd0:BISLPS-XXXXX,fpwd,rpwd,128M,PFS",SCE_CREAT|SCE_RDWR);

例2: パスワードを指定しないで作成する場合
sceOpen("hdd0:BISLPS-XXXXX,,,128M,PFS", SCE_CREAT|SCE_RDWR);

例3: パスワードを指定してオープンする場合
sceOpen("hdd0:BISLPS-XXXXX,fpwd", SCE_RDWR);

例4: パスワードを指定しないでオープンする場合
sceOpen("hdd0:BISLPS-XXXXX", SCE_RDWR);

例5: 読み取り専用パスワードを指定して読み取り専用でオープンする場合
sceOpen("hdd0:BISLPS-XXXXX,,rpwd", SCE_RDONLY);

備考

ファイルシステムドライバでパーティションのフォーマット、マウントを行う前にオープンしたパーティションはクローズしておかないと EBUSYエラーになります。

sceRead
パーティションの拡張アトリビュート領域からの読み込み
形式

#include <sifdev.h>
int sceRead(
	int fd,
	void *buf,
	int count
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 fd   読み出し対象のファイルディスクリプタ 
 buf   読み出したデータを格納するバッファのアドレス 
 count   読み出すデータサイズ(バイト単位で 512 の倍数) 

返り値

成功の場合、読み込んだバイト数を返します。ファイル位置はこの数だけ進められます。0ならばファイルの終わりを意味します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -5   -EIO   入出力エラー 
 -9   -EBADF   fd が有効なオープンされたディスクリプタでない 
 -22   -EINVAL   指定されたサイズが、512 の倍数でない 


解説

すでにオープンされているパーティションの拡張アトリビュート領域から最大 count バイトを buf で始まるバッファへ読み込みます。count に指定できるのは、512 の倍数のみで、それ以外の値を指定した場合はエラーとなります。また EE においては、バッファは必ず 64バイトアラインされていなければなりません。

備考

buf に SPR を指定することはできません。

sceWrite
パーティションの拡張アトリビュート領域への書き込み
形式

#include <sifdev.h>
int sceWrite(
	int fd,
	const void *buf,
	int count
)
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能
マルチスレッドセーフである(割り込み許可状態で呼ぶこと)

引数

 fd   書き込み対象のファイルディスクリプタ 
 buf   書き込むデータが格納されているアドレス 
 count   書き込むデータのサイズ(バイト単位で 512 の倍数) 

返り値

成功の場合、書き込んだバイト数を返します。ファイル位置はこの数だけ進められます。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -5   -EIO   入出力エラー 
 -9   -EBADF   fd が有効なオープンされたディスクリプタでない 
 -13   -EACCES   書き込み権限がない 
 -22   -EINVAL   指定されたサイズが、512 の倍数でない 


解説

buf で示されるバッファーから最大 count バイトまでをファイルディスクリプタ fd によって参照されるパーティションの拡張アトリビュート領域へ書き込みます。count に指定できるのは、512 の倍数のみで、それ以外の値を指定した場合はエラーとなります。バッファは必ず 64バイトアラインされていなければなりません。

備考

buf に SPR を指定することはできません。

HDIOC_FREESECTOR
インストール可能容量の取得
引数

 arg   予約。NULL を指定してください。 
 arglen   arg のサイズ。 
 bufp   インストール可能容量(セクタ単位)を格納する、符号なし 
 32bit 整数へのポインタ 
 buflen   bufp のサイズ。 

返り値

成功の場合、0 を返します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -5   -EIO   入出力エラー 


解説

インストール可能容量を bufp に返します。インストール可能容量とはブラウザが表示するディスクの空き容量と同じ値で、1G 以上の空きは、すべて加算し、1G 以下の 512M、256M、128M はそれぞれひとつまでを加算した容量です。もし、1G 以下の空きが二つ以上あった場合は、その空きを加算するのではなく、その空きより小さいすべてのサイズの空きを加算します。ただし、その時すでにその値と同じ値の空きが加算されていた場合は加算しません。
例えば、512M の空きが二つ、128M の空きがひとつあったとします。
この時、最初の 512M は単純に加算しますが、次の 512M の空きは、256M と 128M の空きとして加算します。次に 128M の空きを見つけてもすでに 128M の空きはひとつ加算されているので、この空きは加算されずに無視されます。

備考

この devctl コマンドが返す値は、上記例が示すように単純な空き容量ではありません。実際に作成できるパーティションの容量より少ない値を返すことがあります。実際に作成できるパーティションの容量は、パーティションのリストと全体の容量から求める必要があります。
1セクタは、512byteです。

HDIOC_IDLE
アイドルモードの設定
引数

 arg   設定値を格納する 8bit 変数へのポインタ 
 arglen   arg のサイズ。 
 bufp   予約。NULL を指定してください。 
 buflen   bufp のサイズ。 

返り値

成功の場合、0 を返します。
エラーの場合、以下のいずれかを返します。
 値   マクロ   結果 
 -5   -EIO   入出力エラー 


解説

ハードディスクドライブがアイドルモードに遷移するまでの時間を設定します。
hdd モジュールは、アイドルモードに遷移する時間をデフォルトで21 分 15 秒に設定しています。設定できる値は以下のとおりです。

 値   意味 
 0x00   timeout disable 
 0x01 - 0xf0   (value * 5) s 
 0xf1 - 0xfb   ((value - 240) * 30) min 
 0xfc   21 min 
 0xfd   Period between 8 and 12 hours 
 0xfe   Reserved 
 0xff   21 min 15 s 


例:
u_char standbytimer = 0xff;
devctl("hdd0:", HDIOC_IDLE, &standbytimer, sizeof(char), NULL, 0);
